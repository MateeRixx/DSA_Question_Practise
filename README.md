âš™ï¸ Git Workflow for Learning

For every new problem, the following workflow is followed to ensure consistent practice in Git:

Checkout: Create a new branch: git checkout -b feature/LC-[Problem_ID]

Solve: Write the solution and tests.

Commit: git commit -m "feat(LC-[Problem_ID]): Solved [Problem Title] using [DSA Topic]"

Push: Push the branch to GitHub.

Merge: Merge the solution back into the main branch.

This practice is key to developing strong habits for team collaboration and project management later on.
